# Module 18: Mock Interview Simulator and Scoring

## Goal
Train under interview-like pressure with structure and scoring.

## Format
- Round 1: Fundamentals (20 min)
- Round 2: Scenario Deep Dive (25 min)
- Round 3: Behavioral and Leadership (15 min)

## Scoring Rubric (0-5 each)
1. Technical accuracy.
2. Practical depth.
3. Risk judgment.
4. Communication clarity.
5. Outcome orientation.

Total score out of 25 per question block.
Target interview-ready score: 18+ consistently.

## Round 1 Question Set
1. Explain container layers and security implications.
2. Why run non-root containers?
3. apt update vs upgrade.
4. Why CVEs remain after patching.
5. Distroless vs slim decision.

## Round 2 Scenario Set
1. Critical CVE in production image with 24-hour SLA. What do you do?
2. Scanner disagreement between Trivy and Grype. How do you resolve?
3. No fix available but internet-facing workload. What controls?
4. Patch caused performance regression in canary. Next steps?

## Round 3 Behavioral Set
1. Convince a delivery team to accept stricter security gates.
2. Describe a time your first remediation plan was wrong.
3. Explain security status to leadership in 2 minutes.

## Real-Time Answer Framework
Use this sequence in every answer:
- Context.
- Risk.
- Action.
- Validation.
- Result.

## Self-Evaluation Sheet
For each answer, write:
- What I explained well:
- What I missed:
- What evidence I can add:
- Final score:

## Live Practice Start
If practicing with an interviewer, start with:
"Question 1: Tell me about a critical CVE you personally remediated in production."

Then run follow-up:
- How did you verify the finding was real?
- Why did you choose that remediation path?
- What was the measurable result?
